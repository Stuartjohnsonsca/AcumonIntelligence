## Page Routes
- / — Home (hero + features)
- /about — About page
- /login — Email + password login
- /login/2fa — OTP verification (6-digit code)
- /login/reset — Forgot password
- /login/reset/verify — Set new password (token from email link)
- /product-access?prefix=X — Auth + subscription check before redirect to subdomain
- /access-denied?prefix=X — No access page with buy option
- /subscribe — Purchase subscriptions (Stripe placeholder)
- /my-account — Tabbed portal (Users/Clients/Subscriptions based on role)
- /my-account/admin — Super Admin panel (Products + Firms)

## API Routes
- POST /api/auth/get-user-id — Returns user ID by email (2FA step)
- POST /api/auth/resend-2fa — Resend OTP code
- POST /api/auth/forgot-password — Send password reset email
- POST /api/auth/reset-password — Reset password with token
- GET/POST /api/users — List/create users (Firm Admin+)
- DELETE/PATCH /api/users/[id] — Delete/update user
- GET/POST /api/clients — List/create clients
- DELETE/PATCH /api/clients/[id] — Delete/update client
- GET/POST /api/subscriptions — List/create subscriptions
- GET /api/products — List all products
- PATCH /api/products/[id] — Update product (Super Admin only)
- POST /api/products/[id]/price-change — Schedule price change
- GET/POST /api/firms — List/create firms (Super Admin only)
- DELETE/GET /api/firms/[id] — Delete/get firm details

## Auth Flow
1. User enters email + password on /login
2. If valid: 6-digit OTP sent to registered email via agents@acumon.com
3. User enters OTP on /login/2fa
4. twoFactorVerified=true set on JWT session
5. Password min length: 8 characters

## Role Hierarchy (highest to lowest)
1. Super Admin (isSuperAdmin=true) — system-wide, all rights + admin panel
2. Firm Admin (isFirmAdmin=true) — manage users + subscriptions for firm
3. Portfolio Owner (isPortfolioOwner=true) — manage clients, assign subscriptions
4. User — access tools for assigned clients with active subscriptions
